3.2 SD-01: UC-00 GitHub OAuth 2.0 로그인

사용자가 [GitHub로 로그인] 버튼을 클릭한 시점부터 세션이 생성되고 Home 화면으로 이동하는 시점까지의 흐름입니다. PKCE(Proof Key for Code Exchange) 방식을 적용하여 인증 코드 탈취 공격을 방지합니다.

4. State Machine Diagram

4.1 AnalysisRequest 상태 머신 다이어그램

stateDiagram-v2 [*] --> PENDING : create(userId, githubId) PENDING --> RUNNING : executeAnalysis() 비동기 실행 시작 RUNNING --> COMPLETED : 파이프라인 전 단계 성공 RUNNING --> PARTIAL : 부분 데이터 수집 완료 RUNNING --> FAILED : 오류 발생 FAILED --> RUNNING : retryFailedAnalysis() [canRetry = true] FAILED --> [*] : 최종 실패 확정 [canRetry = false] COMPLETED --> [*] PARTIAL --> [*]
[그림 4-1] AnalysisRequest 상태 머신 다이어그램

4.2 상태 설명

상태 RequestStatus 설명
대기 PENDING 분석 요청 생성 완료, 큐 대기 중
실행 중 RUNNING 데이터 수집·지표 계산·점수 생성 진행 중
완료 COMPLETED 분석 파이프라인 전 단계 정상 완료
부분 완료 PARTIAL 일부 데이터 누락 또는 최소 분석 조건 충족
실패 FAILED 치명적 오류로 분석 불가

5. Implementation Requirements

Status: Defined in this document (v1.4+)

구현 시 준수해야 할 기술 제약 및 비기능 요구사항:
  • 클래스 설계: 2.3 ~ 2.6절 참고
  • 인증: GitHub OAuth 2.0 + PKCE 방식 적용
  • 데이터 격리: userId 기반 접근 제어
  • 세션 관리: HttpOnly, Secure 쿠키 사용

6. Glossary

용어 정의
AnalysisRequest 분석 요청의 생명주기를 추적하는 엔티티
ActivityData GitHub API에서 수집한 원천 활동 데이터
Metrics 4개 핵심 지표(활동성/다양성/협업도/지속성)
AnalysisResult 분석의 최종 산출물
User GitHub OAuth 2.0으로 인증된 사용자
OAuthClient GitHub OAuth 2.0 API 호출 캡슐화 클래스
AuthenticationService 사용자 인증 및 세션 관리 오케스트레이션
GithubApiClient GitHub REST API 호출 캡슐화
MetricCalculator ActivityData로부터 4개 지표를 산출
CompetencyScorer Metrics를 종합 점수로 변환
FeedbackGenerator 약점 목록으로부터 맞춤형 FeedbackItem 생성
AnalysisService 분석 파이프라인 전체 오케스트레이션
AnalysisRepository AnalysisRequest와 AnalysisResult 영속화
