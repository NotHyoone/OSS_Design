# 기여 가이드 (Contributing Guide)

이 프로젝트에 기여해 주셔서 감사합니다! 아래의 가이드라인을 따라 주세요.

## 브랜치 전략

- `main` : 안정적인 배포 버전
- `develop` : 개발 통합 브랜치
- `feature/<기능명>` : 새로운 기능 개발
- `bugfix/<이슈번호>` : 버그 수정

## 이슈 등록

- 버그 리포트, 기능 요청 등은 GitHub Issues를 이용해 주세요.
- 이슈 제목은 명확하게 작성해 주세요.
- 재현 방법, 기대 결과, 실제 결과를 포함해 주세요.

## Pull Request (PR) 규칙

1. `develop` 브랜치를 기준으로 새 브랜치를 생성합니다.
2. 변경 사항에 대한 테스트를 작성합니다.
3. 모든 테스트가 통과하는지 확인합니다.
4. PR 제목과 설명을 명확하게 작성합니다.
5. 리뷰어를 지정하고 코드 리뷰를 받습니다.

## 커밋 메시지 규칙

```
<타입>: <제목>

<본문 (선택)>
```

### 타입

| 타입       | 설명                          |
|------------|-------------------------------|
| `feat`     | 새로운 기능 추가              |
| `fix`      | 버그 수정                     |
| `docs`     | 문서 수정                     |
| `style`    | 코드 포맷팅, 세미콜론 누락 등 |
| `refactor` | 코드 리팩토링                 |
| `test`     | 테스트 코드 추가/수정         |
| `chore`    | 빌드 설정, 패키지 업데이트 등 |

### 예시

```
feat: 사용자 로그인 기능 추가

- JWT 기반 인증 구현
- 로그인/로그아웃 API 엔드포인트 추가
```

## 코드 스타일

- Python [PEP 8](https://pep8.org/) 스타일 가이드를 따릅니다.
- 함수와 클래스에 docstring을 작성합니다.
- 의미 있는 변수명과 함수명을 사용합니다.

## 테스트

- 새로운 기능을 추가할 때는 반드시 테스트를 함께 작성합니다.
- `tests/` 디렉토리에 테스트 파일을 추가합니다.
- 테스트 실행: `python -m pytest tests/`
